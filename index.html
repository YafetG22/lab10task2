Lab 10 – Framework Web Page

This web page is built and hosted using GitHub Pages, a static hosting framework that allows HTML, CSS, and JavaScript projects to be published directly from a GitHub repository. For Problem 2, I was required to use a web development framework, explain how I installed or set it up, describe how I built this page, and discuss any difficulties I encountered along the way.

Framework Used: GitHub Pages

GitHub Pages is a hosting framework provided by GitHub. It automatically deploys a website from a repository, which makes it easy to use for beginners and does not require a backend server.

How I Installed or Set Up the Framework

1. I created a new public GitHub repository named lab10-framework.
2. I added an index.html file to the repository with HTML and CSS content.
3. In the repository settings, I scrolled to the “Pages” section.
4. Under Source, I selected Deploy from a branch and chose the main branch /root.
5. GitHub Pages automatically built and published the live website.

How I Built the Page

I wrote a simple HTML and CSS layout that includes a title section, paragraph text, headings, and styling. The page explains what GitHub Pages is, how I used it, and how I completed the assignment requirements. All code was written manually without using prohibited frameworks such as Bootstrap or jQuery.

Difficulties Encountered & How I Solved Them

- At first, GitHub Pages did not load because I forgot to name the main file index.html.
- I also had to enable GitHub Pages in the repository settings before the site appeared online.
- There was a short delay before the live site updated, so I learned that each change must be committed and then republished automatically by GitHub Pages.

After fixing the file name and selecting the correct deployment branch, everything worked correctly.
