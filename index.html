Lab 10 – Problem 2

Using the Materialize CSS Framework (Hosted on GitHub Pages)
Framework Used: Materialize CSS

For this assignment, I selected Materialize CSS as my web development framework. Materialize is a modern front-end CSS library based on Google’s Material Design principles. It provides pre-styled components such as cards, grids, buttons, and typography, which makes building visually appealing layouts much simpler.

Materialize is permitted by the assignment guidelines because it is:
• A front-end framework
• Not Bootstrap
• Not jQuery
• Easy to add through a CDN

This allowed me to meet the problem requirements while keeping the setup simple.

How I Installed / Added the Framework

Materialize CSS does not require installation. I added it to my webpage using the official CDN link inside the <head> section of my HTML file. 


<link href="https://cdnjs.cloudflare.com/ajax/libs/materialize/1.0.0/css/materialize.min.css" 
rel="stylesheet">

Once this CDN was included, I could immediately use Materialize components such as:
• container for layout
• card for section formatting
• row and col for responsive grids
• Material icons

No installation, command line tools, or external setup was needed, which made the framework easy to work with.

How I Built the Page

I structured the webpage using Materialize’s responsive grid system and card components. Each major section of the assignment (framework used, installation, building process, difficulties) is placed inside its own card to create a clean and readable layout.

Materialize automatically handles:
• Spacing
• Typography
• Consistent design
• Mobile responsiveness

I added additional custom CSS for body background, card spacing, title styling, and code-block formatting to make the page visually polished and easy to read.

Difficulties Encountered & How I Solved Them

Difficulty 1 – 404 Error on GitHub Pages

The website initially displayed a 404 File Not Found error. GitHub Pages requires the main file to be named index.html. After renaming the file, GitHub Pages successfully recognized and hosted it.

Difficulty 2 – Materialize styling not appearing

When I first loaded the page, none of the Materialize styles applied. I realized I forgot to include the CDN link. Adding it fixed the issue instantly.

Difficulty 3 – Page updates not appearing

GitHub Pages does not update immediately after a commit. I learned that deployment takes around 10–20 seconds. Refreshing the page after waiting solved the issue.

Difficulty 4 – Code block overflow

The CDN line was too long and broke the layout. I fixed this by adding CSS: white-space: pre-wrap; and overflow-x: auto; to ensure code wraps nicely and scrolls if needed.
