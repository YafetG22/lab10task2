Lab 10 – Problem 2

Using GitHub Pages as a Web Development Framework

Framework Used: GitHub Pages

For Problem 2, I used GitHub Pages, a static site hosting framework that allows websites to be deployed directly from a GitHub repository. It is simple, fast, free, and requires no backend server. This makes it an ideal choice for beginners and for hosting lightweight HTML/CSS projects.

How I Installed / Set Up the Framework

GitHub Pages does not require traditional installation. Instead, I followed these steps:

1. Created a new public GitHub repository named lab10task2.
2. Added an index.html file to serve as the website’s main page.
3. Went to the repository’s Settings → Pages.
4. Selected Deploy from a branch as the publishing source.
5. Chose the main branch / (root) folder.
6. GitHub Pages automatically deployed the site online.

How I Built the Web Page

I wrote this webpage fully in HTML and CSS. The design includes:

• A clean centered layout • Section cards with light backgrounds • Smooth fade-in animation • A consistent modern color scheme • Typography styled for readability

This page includes all required explanations from Problem 2 and demonstrates the use of a front-end framework for hosting.

Difficulties Encountered & Solutions

Issue 1: GitHub Pages displayed a 404 File Not Found error.
Cause: The main file was named framework.html instead of index.html.
Solution: Renamed the file to index.html and redeployed.

Issue 2: The site did not update immediately after changes.
Cause: GitHub Pages takes ~10–20 seconds to rebuild.
Solution: Waited for the deployment to complete and refreshed the page.

Issue 3: Unsure where GitHub Pages settings were located.
Solution: Navigated to Settings → Pages and selected the deployment branch.
