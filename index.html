Lab 10 – Problem 2

Using the Materialize CSS Framework (Hosted on GitHub Pages)
Framework Used: Materialize CSS

For Problem 2, I chose Materialize CSS as my web development framework. Materialize is a modern, front-end CSS framework that provides pre-built components, responsive layout grids, and clean visual design—all without using Bootstrap or jQuery. It is lightweight and easy to integrate using a simple CDN link.

How I Installed / Added the Framework

Materialize CSS does not need installation. I added it to my webpage using the official CDN: 

<link href="https://cdnjs.cloudflare.com/ajax/libs/materialize/1.0.0/css/materialize.min.css" rel="stylesheet">

After adding this link inside the <head> section of my HTML page, I was able to use Materialize classes such as containers, cards, buttons, columns, and typography.

How I Built the Page

I built this webpage using Materialize’s components. I wrapped all content inside a Materialize container and used card elements to organize the sections. The framework automatically provides spacing, alignment, visual structure, and responsive styling.

I also added custom CSS to match the assignment style and improve readability.

Difficulties Encountered & How I Solved Them

Difficulty 1: At first, the website showed a 404 error.

Cause: GitHub Pages requires the main file to be named index.html.

Solution: I renamed my file to index.html, and GitHub Pages loaded it correctly.


Difficulty 2: Materialize components were not applying their styles.

Cause: I forgot to include the Materialize CDN link.

Solution: Adding the CDN inside the head section fixed the issue immediately.


Difficulty 3: The site didn’t update right away.

Solution: I learned GitHub Pages takes 10–20 seconds to rebuild after each commit.
